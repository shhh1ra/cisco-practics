## Корпоративный VLAN + Router-on-a-Stick + NAT + ACL

### Задача

Создать и настроить небольшую корпоративную сеть используя VLANы, внутреннюю-VLAN
маршрутизацию, DHCP, NAT (PAT), и политики безопасности, основанные на ACL.

****
# Топология

	                    Сервер
                          |
                    (Внешняя сеть)
                          |
                         R1
                          |
                         SW1
              ____________|_____________
             |            |             |
           VLAN10        VLAN20        VLAN30
          Финансовый      IT          Гостевая
		    отдел      отдел          сеть
           PC1 PC2       PC3 PC4       PC5 PC6

****
# Устройства:

-   1 Router (R1)
-   1 Switch (2960)
-   6 PCs
-   1 Server (Internet simulation)

------------------------------------------------------------------------

# Настройки VLAN

Создайте следующие VLANы на коммутаторе.

| VLAN | Название сегмента | IP адрес        |
| ---- | ----------------- | --------------- |
| 10   | Finance           | 192.168.10.0/24 |
| 20   | IT                | 192.168.20.0/24 |
| 30   | Guest             | 192.168.30.0/24 |
Каждая пара компьютеров должна совпадать с соответствующим VLAN.
****
# Inter-VLAN Routing

Между VLAN коммуникация должна быть пыполнена с использованием технологии **Router-on-a-Stick**

**Требования**:
- Соединение между роутером R1 и коммутатором SW1 должно **trunk**
-   Роутер должен иметь **подинтерфейсы** for each VLAN
-   Каждый VLAN должен иметь **шлюз по умолчанию** на роутере

****
# NAT (PAT)

Роутер R1 должен переводить все внутренние адреса в **один публичный адрес**.

Требования:
- Создать **внутренние интерфейсы** (VLAN подинтерфейсы)
- Настроить **внешний интерфейс** (подключенный к серверу)
- Использовать **PAT (NAT overload)**

Внутренние сети должны быть переведены:
    192.168.10.0 /24
    192.168.20.0 /24
    192.168.30.0 /24

------------------------------------------------------------------------
# Политики безопасности (ACL)

Ввести ограничения доступа между VLAN-ами.

| Отправная точка  | Точка назначения | Политика  |
| ---------------- | ---------------- | --------- |
| Гостевой отдел   | Финансовый отдел | Запретить |
| Гостевой отдел   | IT отдел         | Запретить |
| Финансовый отдел | IT отдел         | Разрешить |
| IT отдел         | Любые сети       | Разрешить |
| Гостевой отдел   | Интернет         | Разрешить |

Требования:
- Использовать **Extended ACL**
- Применить полилитики к **гостевому подинтерфейсу**
- Политики безопасности должны блокировать доступ к внутренним сетям, но должны разрешать доступ в интернет.
****
# Проверка

Сеть считается настроенной когда приведенные тесты успешны:
### Подключение

| Тест                         | Ожидаемый результат        |
| ---------------------------- | -------------------------- |
| Финансовый ↔ IT отделы       | наличие обмена пакетами    |
| Гостевой → Финансовый отделы | отсутствие обмена пакетами |
| Гостевой → IT отделы         | отсутствие обмена пакетами |
| Гостевой → Интернет          | наличие обмена пакетами    |
****
# Команды, которые могут пригодиться при выявлении проблем:

    show ip interface brief
    show vlan brief
    show interfaces trunk
    show ip route
    show access-lists
    show ip nat translations
    show ip nat statistics

# Цель практической работы

Эта практическая работа улучшает знание следующих CCNA концептов:
- VLAN сегментация
- Trunking (802.1Q)
- Router-on-a-Stick
- NAT / PAT
- Access Control Lists (ACL)
- Устранение неполадок в сети
